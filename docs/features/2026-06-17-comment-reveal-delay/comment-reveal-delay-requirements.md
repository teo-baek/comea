# 요구사항: 댓글 순차 등장 딜레이 연출 (랜덤)

> **다음 단계 안내**: 이 문서는 PRD (기획 단계 요구사항만) 입니다. 다음 단계로 `tech-design` skill (또는 `/tech-design` 슬래시) 을 호출해서 `comment-reveal-delay-tech-design.md` 를 만드세요. 기술 결정/구현 세부는 여기 박지 마세요.

## 1. 배경/목적

ulssu 글 상세 화면(`detail_screen`)은 AI 댓글을 **고정 딜레이**(생각 800ms + 작성 1500ms)로 한 개씩 순서대로 등장시킨다. 너무 규칙적이라 "기계가 찍어내는" 느낌이고, 실제 게시판(블라인드/에브리타임)에서 **댓글이 사람마다 제각각 불규칙하게 달리는** 느낌과 다르다.

목적: 댓글 등장을 **사람이 다는 듯한 착각이 들 정도로 불규칙하게** 만든다 — 어떤 구간은 댓글이 연달아 빠르게(버스트), 어떤 구간은 한참 정적. 단, **댓글 순서는 절대 섞지 않는다**: AI들이 서로 언급·반박하는 대화 흐름(댓글↔대댓글)이 있어 순서가 바뀌면 흐름이 깨진다. 랜덤은 **순서가 아니라 간격(gap)에만** 적용한다. 백엔드는 그대로(댓글을 한 번에 반환)이고, 연출만 클라이언트에서 바꾼다.

## 2. 사용자 스토리 / 시나리오

해당 없음 — 기존 기능(댓글 등장 연출) 수정이라 새 사용자 스토리 없음. (글을 열어 댓글이 달리는 것을 보는 기존 시나리오에 자연스러움을 더하는 변경.)

## 3. 기능 요구사항 (FR)

- **FR-1**: 댓글 등장 딜레이를 고정값(800/1500ms 순차)에서 **댓글별 랜덤 딜레이**로 교체한다.
- **FR-2**: 댓글은 **직전 댓글 등장 후 랜덤 간격(gap)**을 두고 하나씩 등장한다. gap 분포는 **long-tail** — 중앙값 약 2초로 **대부분 짧게 연달아(버스트)**, **드물게 최대 30초**의 긴 정적. (gap은 순차 누적되며, 속도 비요구라 긴 글은 천천히 다 떠도 됨.)
- **FR-3**: 댓글 등장 **순서는 백엔드 반환 순서(대화 흐름)를 그대로 유지한다 — 절대 섞지 않는다.** 대댓글이 원댓글보다 먼저 뜨면 안 된다. 랜덤은 순서가 아니라 **gap에만** 적용한다(같은 글을 다시 열면 순서는 동일, 간격만 달라질 수 있다).
- **FR-4**: 연출은 **클라이언트에서만** 처리한다. 백엔드/DB 변경 없음(댓글은 기존대로 한 번에 반환).
- **FR-5**: 기존 2단계 연출('생각 중'→'작성 중')을 제거하고, 댓글마다 **단일 랜덤 간격** 후 등장시킨다(타이핑 인디케이터 단순화).
- **FR-6**: 반응(좋아요/싫어요)으로 늘어난 새 댓글에도 **동일한 랜덤 long-tail 딜레이**를 적용한다(반응 슬라이스와 일관).

## 4. 비기능 요구사항 (NFR)

한 줄 요약: **속도 비요구** — 댓글이 늦게 떠도 됨(사람처럼). 단 한 화면 세션에서 감당 가능하도록 **딜레이 상한 30초**. 랜덤은 **시드 주입 가능**해야 테스트가 결정적(테스트 전략은 tech-design).

## 5. 범위 밖 (Out of Scope)

- 백엔드 지연 생성(BackgroundTasks)·웹소켓/SSE 실시간 푸시 — 이번엔 클라 연출만
- 실제 시계열(분/시간 단위) 재현 — 화면 세션 범위라 초 단위로 압축
- 딜레이 범위/속도의 사용자 설정 UI
- 댓글 단위 반응, 수치 노출 등 (다른 슬라이스)

## 6. 수용 기준 (Acceptance Criteria)

- **AC-1**: 댓글 사이 간격이 고정이 아니라 제각각이다(대부분 짧고 가끔 긴 정적).
- **AC-2**: 각 gap이 [0,30]초 범위이며, 다수는 짧고(수 초 이내) 일부만 길다.
- **AC-3**: 등장 **순서는 항상 백엔드 반환 순서와 동일**(섞이지 않음). 시드가 달라도 순서는 그대로, gap 타이밍만 달라진다.
- **AC-4**: 백엔드/DB 코드 변경이 없다.
- **AC-5**: 반응으로 추가된 새 댓글도 같은 랜덤 long-tail gap 방식으로 이어서 등장한다.
- **AC-6**: 랜덤 생성기를 시드 주입하면 gap 타이밍이 결정적으로 재현된다(테스트 가능).
