# 요구사항: 유저 인증 (최소 — 가입/로그인/로그아웃)

> **다음 단계 안내**: 이 문서는 PRD (기획 단계 요구사항만) 입니다. 다음 단계로 `tech-design` skill 을 호출해서 `user-auth-tech-design.md` 를 만드세요. 기술 결정/구현 세부는 여기 박지 마세요.
>
> **북극성 정합:** `docs/architecture/user-ai-persona-north-star.md` 의 "최소(identity-only)" 슬라이스. §4 전방 호환 규칙을 따른다.

## 1. 배경/목적

comea는 현재 **로그인 없이 익명으로 누구나 글을 쓰고**, 좋아요/싫어요도 익명이다. "내 글", "전체 유저수", "내 행동"이라는 개념이 없어, 목표(유저별 AI 페르소나가 행동으로 진화 — planned.md Phase 3, 북극성 참조)로 갈 토대가 없다.

이 슬라이스는 그 **토대(identity)** 를 놓는다: 이메일+비밀번호로 가입·로그인하고, 글·반응이 "누가 했는지"와 연결된다. 그 결과 ③ 일일 배치가 유저수를 셀 수 있게 되고(Cap 확장 해제), 이후 페르소나 슬라이스가 행동 데이터를 쌓아 올릴 수 있다. AI 페르소나 자체는 이번 범위가 아니다.

## 2. 사용자 스토리 / 시나리오

- **신규 방문자로서**, 이메일+비밀번호로 가입하고 로그인해서 커뮤니티에 글을 쓰고 싶다.
- **로그인 유저로서**, 내가 쓴 글/누른 반응이 내 계정과 연결되어, (앞으로) 나만의 AI가 나를 학습할 토대가 되길 원한다.
- **로그아웃 유저로서**, 글 작성이나 반응을 시도하면 로그인하라는 안내를 받는다.

## 3. 기능 요구사항 (FR)

- **FR-1 (가입)**: 이메일+비밀번호로 가입한다. 이메일은 유일(중복 가입 불가). 비밀번호는 **해시로 저장**(평문 금지).
- **FR-2 (로그인)**: 이메일+비밀번호가 맞으면 **JWT(stateless) 토큰**을 발급한다. 틀리면 인증 실패 응답.
- **FR-3 (로그아웃)**: 클라이언트가 보관한 토큰을 폐기해 로그아웃한다(서버는 stateless).
- **FR-4 (보호된 쓰기)**: **글 작성·반응은 유효한 토큰이 있어야** 가능하다. 토큰이 없거나 유효하지 않으면 401. 토큰의 유저가 글의 작성자/반응자로 연결된다.
- **FR-5 (식별 이음새 — 북극성 §4)**: `users` 테이블 신설 + `posts.author_user_id` / `reactions.user_id`(둘 다 NULL 허용, 기존 익명 데이터 호환) 추가. 신규 글/반응부터 채운다.
- **FR-6 (Flutter 인증 흐름)**: 가입/로그인 화면 제공. 토큰을 **SharedPreferences**에 보관하고, 보호된 요청에 `Authorization` 헤더로 첨부. 로그아웃 시 토큰 삭제. 토큰이 없으면 앱이 로그인 화면으로 유도.

## 4. 비기능 요구사항 (NFR)

- **보안:** 비밀번호는 해시(예: bcrypt류) 저장, 평문/로그 노출 금지. JWT 시크릿은 환경변수(.env). 토큰 만료 설정.
- 한 줄: 응답 지연 요구 없음. 기존 앱 디자인 톤 유지.

## 5. 범위 밖 (Out of Scope)

- 비밀번호 재설정 / 이메일 인증메일 / 소셜 로그인 (대화 중 보류 — 후속 슬라이스)
- 회원 탈퇴 / 프로필 편집
- **AI 페르소나 생성·진화** (중간/풀 슬라이스 — 북극성)
- 글 **읽기(GET)** 권한 제한 — 이번엔 쓰기만 보호(읽기는 앱이 로그인 뒤 노출하되 서버는 비강제). 읽기 강제는 후속 결정.
- 기존 익명 글/반응의 작성자 소급 채움 (NULL로 둠)
- ③ 일일 배치 구현 자체 (이 토대 위 별도 슬라이스)

## 6. 수용 기준 (Acceptance Criteria)

- **AC-1**: 새 이메일로 가입하면 계정이 생기고, 같은 이메일 재가입은 거부된다.
- **AC-2**: 올바른 자격으로 로그인하면 토큰을 받고, 틀리면 실패 응답을 받는다.
- **AC-3**: 토큰 없이 글 작성/반응을 호출하면 401이 반환된다.
- **AC-4**: 토큰을 첨부해 글을 쓰면 그 글의 `author_user_id`가 로그인 유저로 채워진다(반응도 `user_id` 채워짐).
- **AC-5**: 비밀번호는 DB에 평문으로 저장되지 않는다(해시 확인).
- **AC-6**: Flutter에서 가입→로그인→글 작성→로그아웃 흐름이 동작하고, 로그아웃 후 작성 시도는 로그인 화면으로 유도된다.
- **AC-7**: 기존 익명 글(작성자 NULL)도 조회/표시에 문제가 없다.
