# 요구사항: Flutter 반응 버튼 UI

> **다음 단계 안내**: 이 문서는 PRD (기획 단계 요구사항만) 입니다. 다음 단계로 `tech-design` skill (또는 `/tech-design` 슬래시) 을 호출해서 `flutter-reaction-ui-tech-design.md` (기술 설계서) 를 만드세요. 기술 결정/구현 세부는 여기 박지 마세요.

## 1. 배경/목적

comea는 **사람은 글만 쓰고, 댓글은 AI 시민들이 다는 커뮤니티 게시판**이다. 톤은 블라인드·에브리타임·리멤버 같은 일반 게시판 — "끝장 토론 아레나"가 아니다. AI 댓글이 자연스럽게 달리고, 그 안에서 토론이 *덤으로* 일어난다.

백엔드 `POST /api/posts/{id}/reaction`(좋아요/싫어요 → 토론 확장)은 이미 완성됐지만, **Flutter 앱(`comea/`)에는 반응 버튼이 없어** 사용자가 글에 반응해 AI 댓글을 더 끌어낼 수단이 없다. 이 피처는 글 상세 화면에 반응 버튼을 붙여 그 경로를 연다.

서비스 핵심(글쓴이가 관심받는 느낌)에 맞춰, **반응 수치는 노출하지 않는다**(글쓴이 보호). 사용자는 "내 반응이 댓글을 늘렸다"는 효능감을 *수치가 아니라 새 댓글의 등장*으로 느낀다.

## 2. 사용자 스토리 / 시나리오

- **글쓴이로서**, 내 글에 사람들이 반응할수록 AI 시민들의 댓글이 더 붙는 걸 보며, *얼마나 많이 눌렸는지 수치에 시달리지 않고* 관심받는 효능감만 얻고 싶다.
- **독자로서**, 흥미로운 글에 좋아요/싫어요를 눌러 AI 반응을 더 끌어내고 싶다. 개수가 안 보이니 부담 없이 누른다.

## 3. 기능 요구사항 (FR)

- **FR-1**: 글 상세 화면에 좋아요/싫어요 두 개의 반응 버튼을 제공한다.
- **FR-2**: 버튼을 누르면 해당 글의 반응 등록 API(`reaction: "like" | "dislike"`)를 호출한다.
- **FR-3**: 좋아요/싫어요 **개수(누계)는 화면 어디에도 표시하지 않는다.** (글쓴이·독자 모두에게 비노출)
- **FR-4**: 한 사용자가 같은 글에 반응을 **여러 번** 누를 수 있다(취소/토글·1인 1회 제한 없음). 누를 때마다 전송된다.
- **FR-5**: 반응 성공 응답으로 갱신된 댓글 목록을 받아, **기존 댓글은 그대로 두고 새로 늘어난 댓글만** 기존 순차 등장(딜레이) 연출로 이어서 보여준다.
- **FR-6**: 버튼을 누르면 "눌렸다"는 즉각적 상태변화 피드백(색/아이콘 강조 등)을 준다 — 수치 없이도 행동이 반영됐음을 느끼게.
- **FR-7**: 반응 요청이 처리되는 동안 같은 버튼의 **중복 연타 전송을 막고**(처리 중 표시), 응답 후 다시 누를 수 있게 한다.
- **FR-8**: 댓글이 충분히 달린 글(잠김 상태)도 **반응 버튼을 그대로 유지**한다. 별도 "토론 종료" 연출 없이, 게시판에서 댓글이 자연히 안 늘듯 새 댓글이 더 안 달릴 뿐이다.
- **FR-9**: 반응 실패(네트워크 오류/없는 글 등) 시 사용자에게 짧은 오류 안내를 보여주고 버튼을 원상태로 되돌린다.
- **FR-10** (톤 정합): 상세 화면의 "🚨 실시간 끝장 토론" 등 *대결/토론 아레나* 뉘앙스 카피를 **게시판 톤**으로 완화한다(예: 글 종류 표시는 유지하되 과장 문구 제거).

## 4. 비기능 요구사항 (NFR)

- 한 줄 요약: **즉시성 요구 없음.** 반응 후 새 댓글이 곧장 뜨지 않아도 된다(일반 게시판처럼). 기존 앱의 디자인 톤/위젯 스타일을 따른다.

## 5. 범위 밖 (Out of Scope)

- 반응 취소/토글 해제 (대화 중: FR-4에서 "여러 번 허용"으로 결정 → 취소 개념 없음)
- 유저별 1회 제한 / 인증·로그인 (현재 익명, 백엔드도 스택 카운트)
- 댓글 단위(개별 댓글) 반응 — 글 단위만
- 웹소켓/실시간 푸시로 새 댓글 밀어주기 — 별도 "딜레이 연출" 슬라이스
- 좋아요/싫어요 **수치 노출** (원칙적으로 영구 비범위)
- 반응 결과의 통계/분석 화면

## 6. 수용 기준 (Acceptance Criteria)

- **AC-1**: 글 상세 화면에 좋아요/싫어요 버튼이 보이고, 화면 어디에도 반응 개수가 없다.
- **AC-2**: 좋아요(또는 싫어요)를 누르면 잠시 후 새 AI 댓글이 기존 댓글 아래에 순차로 등장한다(없을 수도 있음).
- **AC-3**: 같은 글에 반응을 여러 번 눌러도 매번 전송되며 차단되지 않는다.
- **AC-4**: 처리 중에는 같은 버튼 연타가 중복 전송되지 않는다.
- **AC-5**: 잠긴 글에서도 버튼이 보이고 누를 수 있으며, "토론 종료" 류 배너가 없다.
- **AC-6**: 반응 요청이 실패하면 오류 안내가 뜨고 버튼이 원래 상태로 돌아온다.
